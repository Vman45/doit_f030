7.2 CD-Writer


SCSI CD-Writer am Falcon F030



SCSI CD-Writer werden am Falcon genau wie jedes andere CD-Laufwerk an der externen SCSI2-Buchse angeschlossen. Sollten Sie hier bereits SCSI-Drives angeschlossen haben erleichert das die Sache ungemein.
Es beschränkt sich die Anschlußarbeit auf das richtige Terminieren ihres SCSI-Bus und auf die Installlation der entsprechenden Software.

Zum lesen von fertigen CDs genügt die eventuell bereits vorhandene Software zum Betrieb eines CD-ROMs. Das wird wohl in vielen Fällen das Paket bestehend aus MetaDOS 2.70 und den CD-Tools oder EXTenDOS sein. Es genügt hier das Laufwerk in der hier üblichen Art anzumelden. Sie benötigen nach einen Reboot dann eigentlich nur noch ein passendes Icon auf Ihrem Desktop.
Meiner Meinung nach ist zum lesen von CDs allerdings ein normales CD-Rom wesentlich besser geeignet.
Passende Software zum Betrieb eines CD-Brenners erhalten Sie z.B recht günstig bei Bernd Lohrum der hier ein recht brauchbare Lösung entwickelt hat. Auch bei der Firma Soundpool kann man sich mit Software und auch Komplettlösungen versorgen. Leider fehlt mir hier bis heute jegliche Information dazu.

Bernd Lohrum bietet ebenso anschlußfertige Lösungen für alle Atari-Computer an.
Die Software von Bernd Lohrum, CD-Recorder 2.x unterstützt die CD-Writer folgender Hersteller: 

- Phillips, alle Modelle
- Ricoh, alle Modelle
- Kodak CDD 225
- Yamaha, alle Modelle
- Pinnacle Micro, alle Modelle
- Hewlett Packard

Die Software läuft vollständig unter GEM auf dem Falcon F030 und dem TT.
4MB Ram sollten vorhanden sein, mehr Speicher wäre besser.
Die Kapazität der Festplatte sollte mindestens die Größe der zu beschreibenden CD-R haben. Mehr Platz ist hier natürlich auch besser.

Die nachfolgenden Formate werden im allgemeinen unterstützt 

- CD-ROM
- CD-ROM Multisession
- CD-DA (Digital Audio mit PQ-Editing)
- Mixed Mode CD-ROM
- CD plus

Das zusammenstellen der CD-R passiert im bequemen Drag u. Drop verfahren.
Dateien und Ordner einfach auf die Arbeitsfläche ziehen und das entsprechende Verzeichniss auf der CD-R angeben.

Alle CD-Burner eignen sich genauso für den internen Einbau in einen bereits vorhanden Tower. Hier sollte schon darauf geachten werden, das gilt übrigens für jedes SCSI-Drive gleich am Atari,PC oder Mac, daß die Flachbandkabel zwischen den einzelnen Devices ca.15cm lang sein sollten. Das würde ziemlich genau dem SCSI-Standart entsprechen und die Signalreflektionen innerhalb des Kabels auf ein Minimum reduzieren.
Die meisten CD-Writer wie z.B der Yamaha CDR 102 übertragen durchschnittlich ca. 600Kb/Sec.

Beim Betrieb unter MTasking-Systemen ist man gut beraten wenn man den Rechner wärend des Schreibvorgangs in Ruhe lässt, das gilt nicht nur für den Atari. Selbst die angeblich wesentlich besseren PC- oder Mac-Systeme sind hiergegen nicht immer gut gerüstet. Die beschreibaren CD-Rs sind hier für Fehlschläge einfach noch zu teuer. Es gibt viele Hersteller von beschreibbaren CDs. Hier sollte Sie auf jeden Fall die Datenträger verwenden die auch der Hersteller des CD-Writers angibt.
Gut beraten ist man wenn man sich alle Dateien die man auf eine CD brennen möchte vorher auf einer einzigen Festplattenpartition zurechtlegt und in einem einzigen Rutsch auf die leere CD schreiben lässt.
Laufen weitere Programme, Tasks die momentweise die gesammte Rechnerleistung für sich beanspruchen, kann man mit einer zerschossenen CD rechnen. Genau in diesem Moment setzt der Datentransfer beim Schreibvorgang aus. Das hat natürlich folgen für den eingelegten Rohling.
Bedenken Sie auch beim brennen ihrer eigenen CD das z.B bei der Herstellung einer sog. Multisession CD eine ganze menge Platz auf der CD für die diversen Directorys verbraten wird. Schnell steigt hier der Speicherbedarf der Datenträgerverwaltung auf einige MB an.
500-650Mb an Daten und Programmen passen auf eine CD. Beim Herstellen reiner Sound-CDs können ca. 700-750Mb auf eine CD passen.
Diese CDs sollten stets in einem Rutsch gebrannt werden.

Zum externen Anschluß sollten ausschliesslich 50 polige SCSI-Kabel verwendet werden. Ist Ihr CD-Writer das einzige Gerät am Bus oder ebenso das letzte Device in einer SCSI-Kette sollte die Termination aktiviert werden. Das kann von Hersteller zu Hersteller verschieden Funktionieren. IdR findet bei externen Geräten ein Steckerterminator auf der letzten SCSI-Buchse seinen festen Platz. CD-Writer können auch in ein bereits vorhandenes System eingeschleift werden. Beachten Sie bitte dringlich das das neue Device keine bereits verwendete SCSI-ID belegt.

Bei denen für PC oder Mac erhältlichen Softwareprogrammen ist es möglich den Schreibvorgang im Testmodus auszuprobieren. Hier fehlten mir bisher an der Atari-Software die Möglichkeit für genauere Tests.

Die Stromversorgung eines nackten CD-Writer erfolgt mit der üblichen Steckvorrichtung die auch eben an jedem anderen SCSI-Device vorhanden ist. Beachten Sie hier bitte das ein CD-Brenner ein wenig mehr Strom als eine moderne Festplatte benötigt. Entnehmen Sie keinesfalls die Versorgungsspannungen dem internen Netzteiles Ihres Falcon F030, dieses Netzteil kann die erforderliche Leistung nicht zur Verfügung stellen. Betreiben Sie ihren Falcon in einem Towerumbau der ein PC-Schaltnetzteil enthält kann der Writer hier recht problemlos an diesem Netzteil angeschlossen werden.

Das erstellen von eigenen Audio CDs


Nachdem ich nun endlich mal Zeit hatte, mich mit der CD-Recorder-Software zu beschäftigen, habe ich soeben meine erste Daten-CD mit dem Falcon und dem Yamaha CDR 400T gebrannt. Das Brennen habe ich über ein Image-File auf meiner IDE-Platte in 4-fach-Speed ohne Probleme hinbekommen. Vorher habe ich natürlich Testläufe gemacht, dabei ist mir aufgefallen, daß On-The-Fly-Brennen von der SCSI-Platte Probleme bringt (das wird aber auch von SoundPool gesagt). Ich muß jetzt mal noch andere Varianten ausprobieren, insbesondere die Audio-Sonderfunktionen.

Im großen und ganzen erscheint mir die CDRecoder-Software schon ziemlich ausgereift zu sein. Es sind drei Programme: CDRecorder Pro 2.18 für Daten-, Mix- und normale Audio-CDs, CDRecorder Audio für das Neuzusammenstellen eigener Audio-CDs mit speziellen Eigenschaften (Preßmasterfähig!), CD-Copy zum Kopieren von vorhandenen CDs, dazu ein MetaDOS-Treiber (läuft bei mir auch mit SPIN!0.32) und ein CPX, mit dem man gezielt die einzelnen Sessions bei MultiSession-CDs ansprechen kann.

Ich meine nach meinem ersten Test schon, Du kannst es schon riskieren, den Brenner am Falcon zu betreiben. Ich hätte mit mehr Problemen gerechnet.

Der Betrieb

CD-Recorder sollten immer an staubfreien Orten betrieben werden. Staubpartikel können die CD beschädigen und die Aufnahmen zerstören. Betreiben Sie alle CD-Writer in einem Temperaturbereich von +5 bis +40 Grad. Die relative Luftfeuchte kann dabei bis zu 50% betragen. Können Sie die Temperatur nicht halten ist das Gehäuse zwingend mit einem passenden Lüfter zu kühlen.

Some words about CDRecorder2 from Dennis Vermeire

Hi guys,

There seem to be a few misconceptions here about what exactly one can do with CD Recorder Pro.

SoundPool CD Recorder Pro supports the CD formats CD ROM, Audio CD, Mixed Mode CD and CD extra. Generic formats, such as HFS, PSX, UNIX and various sampler formats, can be copied with the SCSI copy function.

I'll try to explain what the Generic Format do, because there is a lot of confusion about this:
From SCSI hard disk drives and CD ROM drives in ANY format one can burn a CD. After scanning the SCSI bus, all connected SCSI devices are displayed. The drive which has to be copied is selected and one can burn a straight one to one copy to CD from it. What this means in practice is that one can burn a CD from a partition which is NOT accessable by TOS. Mac HFS, UNIX or some special formats used by samplers can all be transferred to CD. The produced CD can NOT be read by TOS, when copying a MAC HFS partition you end up with a MAC HFS CD. So the Generic Format is OK for backup from ALL none TOS partitions and also to copy CDs like the Playstation PSX format. There are a couple of drawbacks: the SCSI Copy function only works like its name suggests with SCSI devices, so IDE drives & partitions are not recognised. The maximum capacity is 650MB, so if one has a partition which is bigger, say 800MB, only the first 650MB are copied.

So far so good... When one wants to create a CD from scratch, this is handled by the the file formatter. The file formatter can NOT handle long file names, these will be truncated and burned like this to the CD. This is a real pain, when working with MagiC or MiNT one starts to use the long file name options rather quickly. The ISO formatter can handle the 9960 level 1 standard, the Atari format, DOS format and a special format which is called: "don't care". With this last option the CD wil be written as the data is. It is not gueranteed however that this CD will work on every OS

Backups Normally one uses the multi session format to backup data on CD ROM. The program supports the archive bit management function. With the "use archive bit" and "clear archive bit" functions the software will only save edited files to the CD. So, when one makes a backup from his data, the next time ONLY data that is new or changed will be written to the CD. This saves a lot of space and time.

Burning Before burning a CD, one can test if the CD will be burned succesfully at the choosen speed. This is a question of knowing your system.... When burning a CD containing 30000 files (fonts, small gifs, HTML pages) the harddisk can't keep up, and 1X is the maximum speed on a TT. When burning a CD containing big files (>1MB) 4X can be sustained easily... If you have enough harddisk space the best system is to make an image, this will take some time (15' max.) but then you can always burn at 4X and the time lost making the image is gained back quickly. You also never have to waist time with the "test" option, because an image will always be written succesfully at maximum speed.

Audio files can be written straight to the CD at 4X without the need to make an image, so copying a Music CD goes very fast. With certain writers 4X can not be handled on a Falcon, but on a TT there are no problems.

I'm not gonna explain the Audio CD functions here, for the time just be happy with the fact that it is one of very few programs on any platform which can prepare a CD ready to be pressed by an Recording Studio. Not suprising really, because the roots from SoundPool is the music business after all.

Comparing CD Recorder Pro with CD Writer is unfair...
CD Writer makes backups from partitions which can then be viewed in TOS. So Minix, HFS, UNIX etc are out of the question. It is capable of making a backup from a accessable partition containing long file names. Accessable partitions is the magic word here... Compared to CD Recorder Pro, CD Writer Gold is very limited and offers only the pure basics. Considering that CD Recorder Pro has come down in price, the choice should be easy....

CD Recorder Pro works with Falcon and TT, with a Link '97 connected ST & STE can also work with the program. For some strange reason the program does NOT work on a Milan (SCSI card ?), I don't know about the Hades...

Cheers from Belgium
